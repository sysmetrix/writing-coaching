훈련 로드맵

12주 3 Phase 집중 훈련

각 Phase를 클릭하면 주차별 드릴과 핵심 미션이 펼쳐집니다.

1
단문 정밀 훈련 — "복문을 단문 2개로 쪼개라"
Week 1–4 · 기초 재설계 · 간결성 6.5 → 8.5 목표
+
WEEK 01
문장 해부 — 절(節) 분리 인식
  • 본인 문서 1편에서 50자 이상 문장 5개 추출
  • 각 문장의 주어-서술어 쌍 개수 세기
  • 쌍이 2개 이상이면 강제 분리 후 재작성
WEEK 02
BLUF 훈련 — 결론 먼저 쓰기
  • 군사 브리핑 기법 BLUF(Bottom Line Up Front) 적용
  • 기존 문단에서 결론 문장을 첫 줄로 이동
  • 이동 전후 독자 이해 속도 비교 자평
WEEK 03
숫자 문장 정제
  • 데이터 수치가 들어간 문장 10개 선별
  • 수치 + 의미 해석을 각각 별도 문장으로 분리
  • 두 문장 연결의 자연스러움 검토
WEEK 04
가독성 자기 채점
  • 1~3주 결과물을 Hemingway 앱에 붙여넣기
  • 목표 Grade Level: 8 이하
  • 8 초과 구간 재작성 반복
Phase 1 핵심 미션
매일 "15자 룰" 적용
핵심 주장 문장을 15자 이내의 단문으로 재작성합니다. 이 한 줄이 요약본, 제목, 슬라이드 헤드라인으로 쓰입니다.
Before: "현재 고립·은둔 청소년의 경우 사회 진입 장벽과 관계 단절이 복합적으로 얽혀 있어 단일 개입보다 연속적 접근이 필요하다"
After: "고립 청소년에게는 단계별 연속 개입이 필요하다"
2
구조 설계 심화 — "목차가 곧 설득 구조다"
Week 5–8 · 구조 내재화 · 의사결정권자용 1p 요약본 정착
+
WEEK 05
PREP vs ORAC 구조 비교 실험
  • 동일 주제를 PREP(주장-근거-예시-주장)으로 작성
  • 동일 주제를 ORAC(목적-근거-행동-확인)으로 재작성
  • 의사결정자 vs 실무자 관점으로 각각 평가
WEEK 06
문서 유형별 템플릿 설계
  • 정책제안서 / 실무협업문 / 기술지시서 3종 분류
  • 각 유형에서 반복되는 섹션 구조 추출·문서화
  • 나만의 '문서 유형 사전' 1페이지 작성
WEEK 07
역방향 아웃라이닝
  • 완성된 본인 문서를 목차만 역으로 추출
  • 목차 흐름이 설득 논리를 따르는지 검토
  • 논리 공백 구간 발견 시 보완 단락 삽입
WEEK 08
1페이지 요약본 훈련
  • 기존 정책문서 1편을 A4 1p 요약본으로 압축
  • 의사결정권자(시장/이사장)가 읽는다고 가정
  • 1p 안에 문제·제안·기대효과·요청사항 4박자 구현
Phase 2 핵심 미션
"문서 이중 납품" 관행 수립
모든 공식 제안서·보고서 작성 시 의사결정자용 1p 요약본 + 실무자용 전문 상세본을 함께 납품합니다.
목표 포맷 — 요약본 4구역
[1] 왜 지금인가 (2문장) · [2] 무엇을 제안하는가 (3줄) · [3] 기대효과 (수치 포함) · [4] 요청 사항 (1줄)
3
독자 분리 전략 — "같은 내용, 다른 글"
Week 9–12 · 독자 전략 · 다독자 반응 실험
+
WEEK 09
독자 페르소나 설정 훈련
  • 자주 쓰는 문서의 독자 5유형 정의
  • 각 유형의 관심사·우려·결정 기준 1줄씩 정리
  • '독자 카드' 한 장 완성 → 집필 전 확인 습관화
WEEK 10
동일 사안 3독자 버전 작성
  • 최근 사업의 핵심 메시지를 청소년용으로 작성
  • 동일 내용을 기관 실무자용으로 재작성
  • 시장/의원용 1p 보고 형식으로 재재작성
WEEK 11
국영문 혼용 정제
  • 기존 문서에서 영어 개념어 전수 추출
  • 대외 문서: 한글 번역 + (영문) 병기 기준 수립
  • 내부 문서: 자유 혼용 허용 기준 명시
WEEK 12
포트폴리오 정리 & 회고
  • 12주 훈련 결과물 모음집 정리
  • Phase별 자기 평가 점수 기록
  • '나의 글쓰기 원칙' 10가지 초안 작성
Phase 3 핵심 미션
"다독자 반응 실험" — 실제 피드백 수집
3버전(청소년·실무자·결정권자)을 실제 해당 대상에게 보내고 이해도·공감도·행동 의향 3항목을 5점 척도로 받아봅니다.

Q3. 이 문서를 읽은 후 무언가 해보고 싶어졌나요? (1~5)

글쓰기 정체성

4가지 시그니처 패턴

문서 분석에서 반복적으로 발견된 원하는님만의 글쓰기 DNA입니다.

PATTERN 01
데이터 → 해석 → 처방
수치 제시 후 즉시 의미 해석, 그다음 정책 처방으로 이어지는 일관된 논증 흐름이 모든 정책문서에서 반복됩니다.
PATTERN 02
조건 목록 + 규칙 기반 지시
기술 문서와 실무 제안서에서 "조건:" 이하에 불릿 목록을 놓고 규칙을 열거하는 방식 — 기술 문서 친화적 사고의 반영입니다.
PATTERN 03
역할-담당자 명시 마무리
문서 말미에 반드시 담당자·연락처·다음 액션을 명시합니다. 문서가 "읽힌 후에도 살아있게" 만드는 실무 설계입니다.
PATTERN 04
위험도·주의사항의 시각 분리
중요 경고를 표 또는 박스로 따로 빼서 강조하는 레이아웃 습관 — 독자의 실수를 예방하는 UX적 글쓰기 감각입니다.
